  - mode="subscription" for membership
- Store Stripe price IDs in env vars:
  - STRIPE_PRICE_PASS_30D
  - STRIPE_PRICE_SUB_MONTHLY
- Create /api/stripe/create-checkout-session that uses logged-in user id and returns checkout url.
- Implement /api/stripe/webhook:
  - Verify signature using STRIPE_WEBHOOK_SECRET and raw body.
  - Handle checkout.session.completed:
    - If payment mode: set access.access_until = now + 30 days; mark order paid; create or update access record.
    - If subscription mode: set subscription_status active; store stripe_subscription_id/customer_id; set subscription_current_period_end; mark order paid.
  - Handle checkout.session.expired -> mark order expired
  - Handle invoice.payment_failed -> set subscription_status past_due (lock access)
  - Handle customer.subscription.deleted -> set subscription_status canceled (lock after period end)
  - Handle charge.refunded or checkout.session.completed with payment_status != paid -> mark refunded and revoke pass access
- Add idempotency: never grant twice for same stripe_session_id; use unique constraint.
- Add /app/billing “Refresh access” button:
  - Calls /api/billing/refresh which looks up the user’s most recent order/session, retrieves session/subscription from Stripe, and re-runs the same grant logic safely (idempotent).
  - This addresses “I paid but it didn’t unlock”.

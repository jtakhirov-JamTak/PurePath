Create a new file client/src/pages/setup-wizard.tsx.

This is a multi-step wizard with 5 steps. The design should be clean and minimal — centered content, one step at a time, with a progress indicator at the top.

The wizard has these steps:

Step 0 — Welcome
  Heading: "Welcome to Leaf"
  Subtext: "Let's set up your daily system using what you created in the workshop. This takes about 10 minutes."
  Show the 5 steps as a preview list (just titles, not forms):
    1. Discovery Profile
    2. Identity Document
    3. Monthly Goal
    4. Starter Habits
    5. First Morning Journal
  One button: "Let's Begin"
  Also a small "Skip setup" link at the bottom that sets onboardingComplete: true and redirects to /dashboard.

Step 1 — Discovery Profile
  Heading: "Your Discovery Profile"
  Subtext: "Enter what you discovered about yourself in Lesson 1."
  Form fields (all textarea, optional — they can fill in what they have):
    - "Core Values" (placeholder: "Your 2 core values and 1 aspirational value, with reasons")
    - "Strengths" (placeholder: "What others consistently recognize in you")
    - "Patterns — Helping" (placeholder: "Top 3 behaviors that serve you")
    - "Patterns — Hurting" (placeholder: "Top 3 behaviors to transform")
    - "Stress Responses" (placeholder: "How you react under pressure")
  Save button that writes to the identity document API (PUT /api/identity-document), storing values in the "values" field, and the rest in a JSON string in the "identity" field.
  "Next" button that saves and advances to step 2.

Step 2 — Identity Document
  Heading: "Your Identity Document"
  Subtext: "Who you're choosing to become. From Lesson 2."
  Form fields:
    - "Vision" (placeholder: "Your 10+ year vision statement")
    - "Identity Statement" (placeholder: "Who you're choosing to become in 1-2 sentences")
    - "Relational Intention" (placeholder: "How you want others to experience you")
    - "Purpose" (placeholder: "What you're contributing and why")
  Save to PUT /api/identity-document with vision, identity (the statement), othersWillSee (relational), purpose fields.
  "Next" advances to step 3.

Step 3 — Monthly Goal
  Heading: "Your First Monthly Goal"
  Subtext: "The goal you set in the workshop. You can refine it later."
  Form fields:
    - "Goal" (placeholder: "What are you working toward this month?") → goalStatement
    - "Why it matters" (placeholder: "Why is this important to you?") → why
    - "Weekly behavior" (placeholder: "What will you do each week to make progress?") → weeklyBehavior
    - "Inner obstacle" (placeholder: "What inside you might block this?") → innerObstacle
    - "IF-THEN plan" (placeholder: "IF [trigger] THEN [small action]") → ifThenPlan1
  Save to PUT /api/monthly-goal with the current month as monthKey.
  "Next" advances to step 4.

Step 4 — Starter Habits
  Heading: "Your Starter Habits"
  Subtext: "Start with 2-3 habits aligned with your identity statement. You can add more later."
  Show a simple form to add 1-3 habits. For each habit:
    - Name (text input)
    - Time of day (select: Morning / Afternoon / Evening)
    - Cadence (default "daily")
  A small "Add another" button (max 3).
  Each habit is saved via POST /api/habits.
  "Next" advances to step 5.

Step 5 — First Morning Journal
  Heading: "Your First Morning Journal"
  Subtext: "Start your daily practice right now."
  Two simple textarea fields:
    - "Gratitude" (placeholder: "What are you grateful for today?")
    - "Intention" (placeholder: "What is your intention for today?")
  Save button posts to POST /api/journals with today's date and session: "morning".
  "Complete Setup" button that saves the journal, sets onboardingComplete: true via PATCH /api/onboarding, and redirects to /dashboard.

Design requirements:
- Use the AppLayout wrapper (same header/nav as rest of app)
- Progress bar at the top showing step X of 5
- Back button on each step (except welcome)
- Clean, centered layout — max-w-2xl mx-auto
- Use existing shadcn/ui components (Card, Button, Input, Textarea, Select, Progress)
- Each step saves independently — if they close the browser and come back, they resume where they left off
- The PATCH /api/onboarding call happens after each successful save to track progress
